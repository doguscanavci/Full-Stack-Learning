# Task: HTML & CSS Static Design Implementation

## Task Description
In this task, I was assigned to complete a partially started web design project. The goal was to recreate a specific UI design (tasarim.png) with pixel-perfect precision using only HTML and CSS.

**Technical Requirements:**
- File Structure: Maintain the integrity of images/, index.html, and index.css.
- Implementation: Follow instructions within HTML/CSS files and build the structure based on the design.

**Checklist:**
- Analyzed the design image.
- Followed HTML & CSS instructions.
- Positioned all images correctly.
- Matched colors, fonts, and sizes with the design.
- Adjusted margins and paddings according to the reference.
- Verified responsiveness and browser compatibility.

---

## My Learning Journey & Reflection
This project marked my first step into Day-to-Day UI development in the third week.

Semantic Architecture: I moved beyond simple div tags and started using header, section, and footer, which makes the code more readable and SEO-friendly.

Visual Fidelity: I mastered using browser DevTools to measure distances and match colors accurately, achieving a result that is nearly identical to the reference design.

Box Model Mastery: Fine-tuning the balance between padding and margin was crucial here to prevent the layout from breaking on different screen widths.

---

- HTML5 (Semantic Tags)
- CSS3 (Flexbox & Box Model)

### Türkçe

# Görev: HTML ve CSS Statik Tasarım Uygulaması

## Görev Tanımı
Yeni başlayan bir çalışma arkadaşınızın yarım kalan web tasarım projesini tamamlamanız isteniyor. Size verilen tasarım görselini (design/tasarim.png), HTML ve CSS kullanarak birebir oluşturmanız gerekiyor.

**Teknik Gereksinimler:**
- Dosya Yapısı: images/, index.html ve index.css dosyalarının düzenini korumak.
- Yapılacaklar: HTML ve CSS dosyalarındaki yönergeleri takip ederek tasarıma uygun yapıyı oluşturmak.

**Kontrol Listesi:**
- Tasarım görseli incelendi.
- HTML ve CSS yönergeleri uygulandı.
- Tüm görseller doğru konumlandırıldı.
- Renkler, yazı tipleri ve boyutlar tasarımla uyumlu hale getirildi.
- Boşluklar (margin/padding) tasarıma göre ayarlandı.
- Responsive tasarım ve tarayıcı testleri yapıldı.

---

## Gelişim Süreci ve Notlarım
Üçüncü haftanın bu ilk gün projesi, görsel bir tasarımı koda dönüştürme becerimi test etti.

Anlamsal Yapı: Sadece div kullanmak yerine header, section ve footer gibi etiketleri kullanarak sayfa yapısını daha profesyonel ve erişilebilir hale getirmeyi öğrendim.

Görsel Hassasiyet: Tarayıcının DevTools (Geliştirici Araçları) özelliğini kullanarak, tasarımdaki renk ve boşluk değerlerini santim santim koda dökme (pixel-perfect) pratiği yaptım.

Kutu Modeli (Box Model): İç boşluk (padding) ve dış boşluk (margin) arasındaki farkı ve bu değerlerin toplam genişliğe etkisini bu projede çok daha iyi kavradım.

---
