### English

# Task: Responsive Challenge

## Task Description
In this task, I was asked to modernize an old, static website to make it mobile-friendly. The main challenge was to implement a fluid layout that adapts to different screen sizes without rewriting the entire HTML.

**Requirements:**
- Use Media Queries to create a mobile-responsive view.
- Implement relative units like rem and em for scalable typography and spacing.
- Utilize align-items properties to manage element positioning in flexible layouts.

---

## My Learning Journey & Reflection
Fluidity over Rigidity: I learned that designing for mobile is not about shrinking elements, but about creating a layout that flows. Switching from px to rem was a crucial step in this realization.
Media Query Logic: This task helped me understand the "breakpoint" strategy—knowing exactly when a layout needs to shift to maintain usability.

---

## Tech Stack
- HTML5 (Semantic)
- CSS3 (Media Queries)
- Relative Units (rem/em)

### Türkçe

# Görev: Responsive Challenge

## Görev Tanımı
Bu görevde, eski ve statik bir web sitesini mobil uyumlu hale getirmem istendi. Temel zorluk, tüm HTML'i yeniden yazmadan, farklı ekran boyutlarına uyum sağlayan akışkan bir düzen kurmaktı.

**Beklenenler:**
- Mobil uyumlu bir görünüm oluşturmak için Media Query kullanımı.
- Ölçeklenebilir tipografi ve boşluklar için rem ve em gibi göreceli birimlerin uygulanması.
- Esnek yerleşimlerde öğe konumlandırmasını yönetmek için align-items özelliklerinden yararlanılması.

---

## Gelişim Süreci ve Notlarım
Sabitlikten Akışkanlığa: Mobil tasarımın öğeleri küçültmek değil, akan bir düzen oluşturmak olduğunu öğrendim. px biriminden rem birimine geçmek bu farkındalıkta kritik bir rol oynadı.
Media Query Mantığı: Bu görev, "breakpoint" (kırılma noktası) stratejisini, yani bir düzenin kullanılabilirliği korumak için tam olarak ne zaman değişmesi gerektiğini kavramamı sağladı.

---

## Kullanılan Teknolojiler
- HTML5 (Anlamsal)
- CSS3 (Media Queries)
- Göreceli Birimler (rem/em)
