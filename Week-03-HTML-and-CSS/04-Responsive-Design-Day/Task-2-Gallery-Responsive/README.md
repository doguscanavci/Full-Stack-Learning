### English

# Task: Gallery Responsive Challenge

## Task Description
The goal of this task was to solve two specific challenges: making a gallery section responsive and conditionally hiding elements based on the screen size.

**Requirements:**
- Convert a 5-column desktop gallery into a 2x2 mobile grid using only CSS.
- Show only the first 4 images on mobile and hide the remaining 6.
- Use advanced selectors like :nth-child() for element filtering.

---

## My Learning Journey & Reflection
Conditional Rendering with CSS: Using :nth-child(n+5) { display: none; } was a breakthrough for me. It showed me how to manage content density for mobile performance without touching the HTML.
Visibility Management: I practiced the difference between display: none (removing from flow) and visibility: hidden (keeping the space), which is essential for precise UI control.

---

## Tech Stack
- CSS3 Grid/Flexbox
- Advanced Selectors
- Responsive Image Management

### Türkçe

# Görev: Galeri Responsive Challenge

## Görev Tanımı
Bu görevin amacı iki spesifik sorunu çözmekti: bir galeri bölümünü responsive hale getirmek ve ekran boyutuna göre öğeleri koşullu olarak gizlemek.

**Beklenenler:**
- Sadece CSS kullanarak masaüstündeki 5 sütunlu galeriyi mobilde 2x2'lik bir ızgaraya dönüştürmek.
- Mobilde sadece ilk 4 resmi gösterip kalan 6 resmi gizlemek.
- Öğe filtreleme için :nth-child() gibi gelişmiş seçiciler kullanmak.

---

## Gelişim Süreci ve Notlarım
CSS ile Koşullu Gösterim: :nth-child(n+5) { display: none; } kullanımı benim için bir dönüm noktası oldu. HTML'e dokunmadan mobil performans için içerik yoğunluğunu nasıl yöneteceğimi öğrendim.
Görünürlük Yönetimi: Hassas UI kontrolü için temel olan display: none (akıştan çıkarma) ve visibility: hidden (alanı koruma) arasındaki farkı uygulama fırsatı buldum.

---

## Kullanılan Teknolojiler
- CSS3 Grid/Flexbox
- İleri Seviye Seçiciler
